Optimización SEO y Accesibilidad

Objetivos RA1 y Prueba P2 (Portfolio)

  • Recordatorio: cada uno trabaja en su servidor real vía VS Code + GitHub.

Optimización SEO básica

Etiquetas Meta esenciales

Etiqueta Uso recomendado
<meta charset="UTF-8"> Define la codificación de caracteres. Siempre al inicio del <head>
<meta name="viewport" content="…"> Hace tu sitio responsive en móviles.
<title> Título de la página (50–60 caracteres).
<meta name="description" content="…"> Resumen atractivo (150–160 caracteres).
<link rel="canonical" href="…"> Evita contenido duplicado indicando tu URL preferida.

Ejemplo completo en <head>

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Portfolio • Ana Pérez – Front‑end Developer</title>
  <meta name="description" content="Portafolio web de Ana Pérez, desarrolladora front‑end especialista en HTML5, CSS3 y JavaScript.">
  <link rel="canonical" href="https://www.anaperez.dev/">
</head>

Tips

  • Title: colocar palabra clave principal al inicio, seguido de nombre o marca.
  • Description: incluir un llamado a la acción (p.ej. "Descubre mis proyectos…").

Encabezados y estructura semántica

  • Un solo <h1> por página: título principal.
  • Usa jerarquía lógica:
<h1>Portafolio de Ana Pérez</h1>
<h2>Sobre mí</h2>
<h3>Formación</h3>
<h2>Proyectos</h2>
<h3>Proyecto A</h3>

Aprovechar etiquetas semánticas:

<header>…</header>
<nav>…</nav>
<main>…</main>
<section>…</section>
<article>…</article>
<aside>…</aside>
<footer>…</footer>

(Esto ayuda a los buscadores a entender el contenido.)

URLs amigables

  • Solo minúsculas, sin espacios.
  • Palabras separadas por guiones (-).
  • Evitar parámetros innecesarios.

Ejemplos correctos

https://www.tu-dominio.cl/about.html
https://www.tu-dominio.cl/projects.html#contact

Robots.txt

Define qué puede indexar Google y otros motores:

# /robots.txt
User-agent: *
Allow: /

Sitemap: https://www.tu-dominio.cl/sitemap.xml
  • Colocar este archivo en la raíz del servidor (/robots.txt).
  • El Sitemap facilita que los bots encuentren todas las páginas.

Sitemap.xml

Un "mapa" en XML de las URLs:

<?xml version="1.0" encoding="UTF-8"?>
<urlset 
  xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://www.tu-dominio.cl/</loc>
    <lastmod>2025-04-21</lastmod>
    <priority>1.00</priority>
  </url>
  <url>
    <loc>https://www.tu-dominio.cl/about.html</loc>
    <lastmod>2025-04-20</lastmod>
    <priority>0.80</priority>
  </url>
  // Más URLs aquí 
</urlset>

Subir como /sitemap.xml.

Actualizar lastmod cada vez que se cambie la página.

Imágenes y atributos ALT

Nombre de archivo descriptivo:

foto-perfil-ana-perez.jpg

alt descriptivo:

<img src="foto-perfil-ana-perez.jpg"
     alt="Ana Pérez sonriendo con fondo de oficina">

srcset para responsive:

<img
  src="foto-800.jpg"
  srcset="foto-400.jpg 400w, foto-800.jpg 800w"
  sizes="(max-width: 600px) 400px, 800px"
  alt="Ejemplo de proyecto A">

Enlaces internos y texto ancla

  • Usar texto claro: "Más sobre mis proyectos" en lugar de "haz clic aquí".
  • Estructura de navegación: un <nav> con enlaces a las secciones clave facilita el rastreo.
<nav>
    <ul>
      <li><a href="/">Inicio</a></li>
      <li><a href="/about.html">Sobre mí</a></li>
      <li><a href="/projects.html">Proyectos</a></li>
      <li><a href="#contact">Contacto</a></li>
    </ul>
</nav>

Performance y Core Web Vitals

Aunque no es SEO puro, la velocidad influye en el posicionamiento:

  • Optimizar imágenes (WebP, compresión).
  • Minificar CSS/JS: usar herramientas como terser o cssnano.
  • Habilitar caché en el servidor (headers Cache-Control).
  • Revisar con Lighthouse: DevTools → Lighthouse → Genera reporte → pestaña "Performance" y "SEO".

Herramientas de validación

  • HTML/CSS → W3C Validator
  • Lighthouse → Chrome DevTools → Audits → SEO
  • WebAIM Contrast Checker → contraste de texto vs. fondo
  • Search Console (opcional) → monitorea errores de rastreo

Resumen de implementación

  1. Metadatos esenciales:

    <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Portfolio • Nombre Apellido</title>
    <meta name="description" content="Portafolio web de Nombre Apellido, desarrollador front‑end">
    <link rel="canonical" href="https://tu-dominio.cl/">
</head>
  2. Etiquetas de encabezado: un solo <h1> por página, estructura jerárquica.
  3. URLs amigables: /about.html, /projects.html.
  4. Mapa de sitio: crea sitemap.xml y sube al servidor.

  5. Robots.txt:

    User-agent: *
Allow: /
Sitemap: https://tu-dominio.cl/sitemap.xml
  6. Práctica: añadir todos los elementos al repositorio, validar con DevTools → Lighthouse → SEO.

Mejora de accesibilidad

  • Añadir lang="es" en <html>.
  • Roles ARIA y landmarks:
<header role="banner">…</header>
<nav role="navigation">…</nav>
<main role="main">…</main>
<footer role="contentinfo">…</footer>
  • Texto alternativo (alt) descriptivo en imágenes.
  • Contraste: revisa colores con WebAIM Contrast Checker.
  • Prueba de navegación con teclado (Tab, Shift+Tab).
  • Ejercicio práctico: analiza tu portfolio y corrige al vuelo.

Validaciones JS avanzadas

  1. Validar correo y campos (uso de expresiones regulares):

    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
if (!emailRegex.test(email)) {
  showError("Email no válido");
}
  2. Longitud y caracteres: mínimo/máximo, no caracteres especiales.

  3. Modularizar validaciones:

    function validarCampo(id, fnValidacion, msgError) { … }
validarCampo("name", v => v.trim() !== "", "Nombre es obligatorio");
validarCampo("email", v => emailRegex.test(v), "Email inválido");
  4. Práctica: mejora tu formulario de contacto con estas funciones en tu servidor.

Feedback dinámico & UX

  • Mostrar mensajes en línea en lugar de alert().
  • Uso de classList para togglear estados:
const msg = document.getElementById("feedback");
msg.textContent = "Enviado con éxito";
msg.classList.add("visible");
setTimeout(()=> msg.classList.remove("visible"), 3000);
  • Ejercicio: implementa un "toast" o notificación modal en tu portfolio.
  • Prueba en móvil y escritorio.
