Uso de Postman

Clase práctica: Usando Postman para probar tu API

En esta clase aprenderás a usar Postman, una herramienta súper útil para probar tus APIs sin necesidad de tener un sitio web funcionando. Es como hablar directamente con el "cerebro" del sitio.

¿Qué necesitas?

  • Tener Postman instalado (descargar aquí)
  • Tu servidor local corriendo (por ejemplo, XAMPP, Laragon o similar)
  • Tu archivo api.php funcionando (del ejemplo anterior)

Paso 1: Abrir Postman

Abre Postman. Verás una ventana parecida a un navegador, pero con herramientas para hacer peticiones.

Paso 2: Crear una nueva petición

  1. Haz clic en New o + New Tab.
  2. Selecciona el tipo de petición como GET.
  3. En la barra de URL, escribe la dirección de tu API. Por ejemplo:
    http://localhost/proyectos-api-db/api.php
  4. Presiona el botón Send.

¡Listo! Si todo está bien configurado, verás abajo una respuesta como esta:

[
  {
    "id": "1",
    "nombre": "Calculadora Web",
    "imagen": "../../assets/imgs/semanas/semana10/proyecto1.png",
    "descripcion": "Una calculadora simple hecha en JavaScript.",
    "url_github": "https://github.com/usuario/calculadora",
    "url_produccion": "https://usuario.github.io/calculadora"
  },
  ...
]

¿Qué significa esto?

  • GET: Es una forma de decir “quiero leer algo”.
  • Postman está pidiendo datos a tu API.
  • La API responde con información en formato JSON, que los sitios web pueden leer y mostrar.

Tip adicional

Si ves errores, revisa lo siguiente:

  • ¿Está corriendo XAMPP o tu servidor?
  • ¿Escribiste bien la URL?
  • ¿Tu archivo api.php está en la carpeta correcta?

Actividad práctica

  1. Prueba la API del portafolio con Postman.
  2. Asegúrate de que la respuesta sea correcta.
  3. Puedes agregar más parámetros para probar el comportamiento de la API.

Recuerda

En tu proyecto final de portafolio, debes tener tu API funcionando correctamente y podrás usar Postman para probarla antes de integrarla en el sitio web. ¡Usar Postman es como tener un "visor de energía" para tu backend!
